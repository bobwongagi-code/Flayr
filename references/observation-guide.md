# Flayr 视频观察指引

> 本文是 LLM 看视频时的"观察方法"指引。
> 与 `structure_library_full.md`（结构层）、`ANALYSIS-PROMPT.md`（流程层）并列，
> 在 `analysis_input.md` 装配时一起进入 LLM 输入。
>
> 适用对象：短视频带货分析（重点：东南亚 TikTok Shop）
> 加载方：`flayr_core/prompt.py:write_analysis_input`

---

## ⚠️ 宪法：S1-S6 看功能，不看模式

这一条优先级高于本文其他所有规则、`structure_library_full.md` 的模块定义、
和 `ANALYSIS-PROMPT.md` 的流程描述。

> **结构库的 32 个模块是"已知功能的常见实现方式列举"，不是穷举。**
> 当一段视频"既不像 S1-A 也不像 S1-B"时，
> 先问这段在完成什么**功能**，再判断它属于哪个槽位。
> 模块编号识别不出来就写 `unknown`，**不要强行套**。

S1-S6 的功能定义（不是实现方式）：

| 槽位 | 功能 | 不是 |
|------|------|------|
| S1 Hook | 抢注意力（让用户不划走） | 不一定是"产品没出现的前几秒" |
| S2 产品引出 | 把注意力过渡到产品 | 不一定是"产品首次出镜的瞬间" |
| S3 使用过程 | 展示怎么用 | 不一定是中段的镜头 |
| S4 效果呈现 | 展示用了之后怎样 | 不一定是前后对比 |
| S5 信任放大 | 让用户相信 | 可跳过，但跳过要符合条件 |
| S6 CTA | 推动下单 | 不一定是结尾的最后几秒 |

**判断阶段归属时，永远先问"这段在完成什么功能"，再问"它用了结构库哪个模块"。**

---

## 一、整片观察重点（最重要）

### 1.1 段切方法

不预设某秒数对应某阶段。**综合画面、口播、字幕、BGM 四轨**，看**信息变化拐点**来切段。

切完后回问每一段：

1. 这段在完成什么功能？
2. 这个功能对应 S1-S6 的哪个槽位？
3. 段内用的是结构库哪个模块？识别不出来写 `unknown`。

**警告：不要用以下方式切段**：

- ❌ 按"产品首次出镜时间"决定 S1→S2 边界（很多达人开场就拿着产品）
- ❌ 按"节奏拐点"判断 S3 推进（部分市场视频节奏天然平稳，拐点几乎不存在）
- ❌ 按"视觉峰值"等同 Hook 时段（口碑截图小窗也可能是 Hook 载体）
- ❌ 按"标杆视频的阶段时间"映射到达人视频（两条视频时长可能差 3 倍以上）

### 1.2 整体分析的两层重点

#### 第一层（必做）：产品-人-场景三角

在做任何 stage 分析前，先建立三角理解：

- **用户群体**：基于产品 + 视频内容推断（视频里有谁、对谁说、用什么语气）
- **卖点**：产品提供的核心价值
- **痛点**：用户的什么具体困扰被这个产品解决
- **痛点场景化展示方式** ← **最关键**：达人用什么场景把痛点演给用户看？

这一层决定了所有后续判断的基准。没建立三角就开始分段，等于在没坐标系的纸上画曲线。

#### 第二层（重要）：视角和互动方式

- **拍摄视角**：达人正面出镜对话 / 第一人称手持 / 旁观纪实 / 生活化拼接 / 用户视角代入
- **互动方式**：直接镜头说话 / 演示+点评 / 纪实+旁白 / 让用户代入角色

> 例：亲子品类的视频画面是快递到家拆包装、孩子好奇凑过来——这是"生活化纪实视角 + 让用户代入家长"，
> 达人不出镜也不影响信任建立。**不要因为达人没出镜就扣分**。

视角和互动方式不影响功能判断，但影响**执行性差距**的判断：
同样是 S2，达人可能用对话视角讲产品，标杆可能用纪实视角让用户自己代入——
**两种都可能完成 S2 的功能，差距分析时要先识别视角差异再判断哪种更适合该品类**。

## 二、抽帧观察框架（per-frame）

对每张关键帧，按以下维度提取信息。**不要自由发挥**，按维度走避免漏要素。

### 2.1 基础维度

- **视觉主体**：人物 / 产品 / 场景 / 文字
- **焦点位置**：中心 / 边缘 / 全屏
- **镜头语言/取景完整性**：画面是否歪斜、动作主体是否完整入镜（如演示对象只拍到一半）、
  关键动作是否在画面内完成——这是 S3 使用过程**执行性差距**的有效信号
  （实证：达人刷马桶画面歪斜、马桶只见一边，标杆全貌完整，体验差距明显）
- **遮挡与 UI 危险区**：关键动作/文字是否被手、头发、道具遮挡；关键内容是否落在
  TikTok UI 遮挡区（画面底部购物车区、右侧点赞按钮区）——物理存在 ≠ 有效传递，
  落入遮挡区的内容消费者实际看不到
- **动作类型**：静态展示 / 使用过程 / 反应表情 / 前后对比
- **可读字幕**：品牌 / 价格 / CTA 文案 / 卖点叠字
- **与口播的关系**：同步 / 提前 / 滞后 / 无关（音乐占位）

### 2.2 强调信号维度（重点）

这些是**达人主动强调**的信号，权重高于普通画面：

- **字幕样式变化**：字号放大、变色、emoji 标记 → **必是重点**
- **画中画**：小窗内容（认证截图、营养元素图、口碑评论截图、用户对话截图）→ **小不代表不重要，通常是强调**
- **达人面部表情变化**：惊讶、皱眉、点头、笑容 → 情绪节奏的关键锚点
- **特殊音效**：点击声、转场音、强调音（与 BGM 区分）→ 强调标记

> 当画面里同时出现"字幕高亮 + 画中画 + 表情变化 + 特殊音效"中任意 2 个以上 →
> 这帧承载的信息一定是该段的核心，不要把它当背景。

## 三、四轨对齐

模型不能只看单帧。看的是**四轨之间的对齐与错位**：

| 轨 | 数据源 |
|------|------|
| 画面 | 关键帧序列 |
| 口播 | `transcript.srt` 的时间戳分段 |
| 字幕 | 可读叠字（从画面提取） |
| BGM | 在场 / 缺席 / 风格变化点（见第四节）|

**对齐分析的三种情况**：

1. **四轨同步强调同一信息** → 这是强信号点（如：字幕高亮"现在下单"+ 口播喊 CTA + 画面指向购物车 + BGM 推到高潮）
2. **轨道之间错位** → 注意是技巧（蓄势）还是漏洞（混乱）
3. **某一轨独自承载信息**（如只有字幕说但口播没说）→ 在 stage_analysis 里要明确标 `voice_only` / `visual_only`

## 四、BGM（单列）

BGM 是**非对称信号**：好不加多少分，坏扣很多分。判断时按这个不对称权重处理。

### 4.1 加分场景（权重低）

BGM 风格与产品目标人群、使用场景的情绪基调相容。
（例：亲子品类配轻快童趣旋律、健康品类配舒缓自然音乐、运动品类配节奏感强的节拍）

### 4.2 扣分场景（权重高，直接影响 severity）

- **缺失**：本应有 BGM 衬托的场景（如生活化纪实、情绪铺陈）完全无音乐
- **破坏**：BGM 节奏 / 音量与画面节奏强烈冲突（如静态展示配高速电子节拍）
- **严重错配**：BGM 风格与产品目标人群、情绪基调强烈冲突
  （例：亲子内容配激烈摇滚或恐怖配乐、健康养生品类配电子舞曲）

**严重错配应直接把所属 stage 的 severity 升级到 `large`**，并在 gap_summary 里点明。

### 4.3 中性场景

BGM 平凡但不破坏 → 不加分也不扣分，gap_summary 里不必提及。

## 五、失误模式预警（自检清单）

完成 stage_analysis 输出前，对照以下清单自检。命中任何一条 → 重新审视相关 stage。

### 5.1 阶段判断失误

- ❌ 把"标杆 CTA 的位置（如 27~31s）"映射到达人视频固定位置 → CTA 位置必须按达人视频内容找
- ❌ 因为"产品出现早"就判定没有 Hook → Hook 可能就是"产品 + 强口播 + 痛点直击"
- ❌ 因为"节奏平稳"就判定结构松散 → 部分市场视频节奏天然平稳，看**信息变化拐点**而非节奏变化
- ❌ 把"视觉峰值"等同于"Hook 时段" → 口碑截图、用户证言小窗也可能是 Hook 载体

### 5.2 证据判断失误

- ❌ 把口播说的认证（如"经过 XX 审批"）当成画面有认证 → 必须 `support_status = voice_only`
- ❌ 把音乐占位（如 `*outro music*`）当成有效口播
- ❌ 漏看口碑截图小画面 → 小画面承载的信任 / 卖点信号比看起来重要
- ❌ 漏看视频后半段 → 长视频特别容易漏，必须均匀覆盖全片

### 5.3 视角与品类失误

- ❌ 因为达人没出镜就扣分 → 纪实视角 / 用户代入视角同样有效
- ❌ 把通用规则套到特定品类 → 例如"亲子内容必须配童趣 BGM"不是规则，是参考；
  品类的真实判断要基于"产品目标人群 + 痛点场景 + 当地市场习惯"综合
- ❌ 对达人执行的本地化方式做想当然评价 → 不熟悉的市场习惯先描述、再判断，
  不要直接说"达人做错了"
