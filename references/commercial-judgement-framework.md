# Flayr 带货短视频商业评判框架

> 适用对象：东南亚 TikTok Shop 带货短视频对比分析。
> 本文是商业判断层，优先解决“什么差距真正影响 GMV”。
> 它不替代 `ANALYSIS-PROMPT.md` 的流程和 schema 约束，也不替代
> `structure_library_full.md` 的模块定义。

## 核心标尺

唯一评判标准是对转化/GMV 的贡献。标杆 GMV 更高是前提，但分析不是比较“像不像标杆”，而是找出标杆中真正驱动转化、达人缺失或做弱的有效动作。

- 形式不同但都达成功能，按风格差异处理，不判差距。
- **"都做了但呈现单薄"上限 medium**（2026-06-13 判例）：双方均完成该阶段功能、达人仅呈现
  单薄（场景单一/缺效果对比/表现力弱）时，最重判 medium；large 必须是达人侧缺失或基本无效
  （敷衍/晃动/纯口头无验证），或标杆动作演示 vs 达人完全口头。标杆素材质感超规格
  （"欺负达人"）不构成把达人压成 large 的理由。
- 达人某点优于标杆，明确记为亮点，不判达人差距。
- 双方都没有的维度，标“均未涉及”，不硬找差距。
- 对比不计视频时长，按功能和商业作用判断。

## 品类自适应权重

分析前先判断产品三属性，并据此调整权重：

- 客单价：低 / 中 / 高。客单越高，CTA 权重越低，卖点和背书权重越高。
  - **判定口径（运营填，2026-06-13）**：以 **TikTok Shop 同品类**为参照（不是货架电商/线下零售，
    也不是绝对金额）——low=该类目走量/入门款，mid=主流款，high=类目内溢价/高端/专业级定位
    （品牌溢价、高规格）。客单价是**相对概念**：同是手机壳，杂牌是 low、贝尔金/图拉斯是 high。
    来源优先级：运营经 `--product-tier` 提供（领域知识，最可靠）> 模型世界知识判断（兜底，
    对本地品牌价位常无谱）。运营价格经 `--product-price` 留痕，供审计判档依据。
  - **冗余性实证（6 高价样本：are_xie 保健品 + poboji 榨汁机 ×5）**：在 `considered` 决策门槛下
    price_tier 对推导零影响——"决策有多重"已被 decision_threshold 接走。唯一非冗余点在
    `impulse` 分支：`impulse+high`（贵但冲动买）会被误当低价冲动品（背书权重 0.6 偏低），
    代码已加告警触发器，命中即人工复议。此组合在 Flayr 选品场景下结构性罕见，故 price_tier
    暂列低优先观察项，不立独立原型。
- 决策门槛：冲动可买 / 需被说服。门槛越高，卖点、效果验证和背书权重越高。
- 驱动类型：情绪 / 功能理性。情绪驱动品类上调表达调性、BGM 和场景氛围权重；功能理性品类优先看卖点链和效果验证。

Hook 恒为高权重，因为停留是后续转化的前提，任何品类不降权。

## 阶段判定

阶段按内容功能判断，不按秒数和出场顺序。
结构先验（仅辅助，功能判断永远优先）：S1 Hook 与 S6 CTA 的位置相对固定（开头/结尾），
S2-S5 中段内容是浮动的——认证、卖点、演示可以出现在任何位置，按功能归属，不按出现位置。

- S1 Hook：用户划走前是否完成注意力抢夺和代入。
- S2 引出：产品引入是否自然，是否为后续卖点铺垫；Hook 已强调的点不重复扣分。
- S3 演示：只判断用户能否看懂“怎么用”。
  - **使用过程判例（2026-06-12）**：标尺是"关键环节覆盖"，不是全程完整性——重点动作有
    （如涂抹有、推开镜头有）即算到位；过长的全程记录不加分，需重新审视。
  - **画面晃动判例（2026-06-12）**：晃动到观众抓不住重点/看不清情况 = 无法有效接收，
    按"基本无效"处理（观众视角可看性，非镜头美学）。
- S4 效果验证：判断用户是否看到并信服产品有效，包括使用结果、产品自身卖点效果、香味/质感/口味等感官体验。
  使用后**效果是否突出、有无对比**是 S4 到位与否的核心判据（2026-06-12 判例）。
- S5 叠 buff：判断是否有“有支撑的”第三方背书。**第三方背书 = 机构类型 + 关联性门槛同时成立**：
  - 机构类型（泛化到类型，不限具体名单）：监管/认证机构（如 KKM、Halal、SIRIM、TISI）、
    行业协会、第三方评测中心/实验室、高校与研究机构、三方调研咨询公司、疾病·医院·防治中心。
  - 关联性门槛（**关键**）：该机构的实验结果 / 数据报告 / 研究**在证明这个产品的价值**，才算背书。
    例：“某大学测试证明本产品 2.5 分钟灭活 99.98% 病毒” = 背书；
    仅提到机构名字、赞助商、合作方 logo，而无证明本产品价值的数据/结论 = 不算背书。
  - **市场准入基线 ≠ 差异化背书**（2026-06-11 判例）：清真（Halal）认证在穆斯林市场是默认门槛，
    不与权威认证（KKM 等政府认证、检测报告、临床数据）同层级；达人仅有基线合规而标杆有
    权威认证 = 背书层级差，按背书缺失处理（are_xie 判例：达人口播 Halal vs 标杆 KKM 可视化 → large）。
  - 纯口播自述功效、无第三方支撑的声明都不是背书。无强背书品类看**信任类**购买理由（凭什么信这个产品）是否立住。
  - **S5 只装信任内容**：产品价值信息（数量、容量、耐用性、价格划算感、性价比量化）是**卖点链内容，归 S2-S4，不归 S5**——标杆讲"6+6 刷头够用 3 个月很划算"属卖点信息，缺失算 S2 卖点链差距，不算 S5 背书差距。
  - 品类条件判例：口服/健康等**高决策门槛品类**，权威认证（KKM 等）的**视觉化**背书缺失或明显弱于标杆（标杆图文展示 vs 达人仅口头提及）→ 按 large 处理；低客单价冲动品类双方均无背书 = 正常状态，判"均未涉及，small"。
- S6 促单：判断是否自然带到明确下单。给**信任类**理由归 S5，给指令归 S6；划算感/价值量化等卖点理由归卖点链（S2-S4）。两条判例：
  - **CTA 过度重复 = 硬推，应扣分**：反复喊"快来买/官方/直发"会让整条视频显得硬推，尤其当
    S1-S4 没吸引住人、没讲清卖点时，CTA 越多越反效果；标杆的好 CTA 是卖点链讲顺后顺水推舟的一句。
  - **弱 CTA ≠ 有效 CTA**：一句含糊带过、观众无感的提及不算完成促单功能；判断有无之外还要判强度。
    敷衍档（2026-06-11 kakwan 判例，马来语母语者三遍重听才确认的一句轻带 CTA）：形式上有但
    极短、敷衍、不注意听不到。单侧执行分标尺：0=不执行，0.5=敷衍，1=合格，2=好；
    差距 = 标杆执行分 − 达人执行分，由代码相减，两侧打分各自独立、不交叉对比。

## 两条贯穿维度

- 卖点链（S2→S5）：看说服链是否层层递进、有无断点；结合世界知识和市场背景判断主打卖点是否选对。
  - **卖点质量判断（品类优先级）**：不同品类的消费者决策优先级不同。判断卖点时，先问"这个品类的目标消费者最在意什么"，再看达人和标杆各自主打的卖点是否命中了高优先级。例：儿童牙膏的妈妈最在意牙齿健康/防蛀/安全成分，其次是使用体验/口味，"省钱/节省"属于低优先级；达人讲了"节省"不等于在功能卖点上完成任务，标杆讲了"防蛀/健康"则命中了核心决策因素。卖点质量判断必须在对比前完成，不能只比"谁讲了卖点"而不比"讲的是不是对的卖点"。
  - **卖点选择偏差 ≠ 达人完胜**：达人讲了一个卖点、标杆也讲了一个卖点，不代表双方持平。如果达人的卖点在品类决策优先级中排位低，而标杆的卖点排位高，这本身是一个 medium 级差距——达人没有抓住消费者最关心的购买理由。
  - **省钱卖点的隐含前提**：省钱/划算逻辑只对"日常已消费/刚需补货"动机成立；非刚需品（如保健品）在没讲清产品价值之前就谈省钱，等于默认观众已在日常消费该品类，是卖点选择偏差的典型形态。
- 表达调性匹配：看口播风格、BGM、场景、本地语言地道度。情绪品类权重上调，功能理性品类不得让调性压过 Hook 和卖点链。

## 判断顺序

每个维度必须按以下顺序判断：

1. 识别该维度对转化有贡献的关键动作。
2. 判断达人和标杆谁做到、谁更有效。
3. 先写核心结论：谁强、强在哪个动作、为何影响转化。
4. 再据此定优先级和 severity，结论与定级必须一致。
5. 标杆有效达人缺失，按权重定级。
6. 达人有效标杆弱，记亮点，不判达人差距。
7. 都有效仅形式不同，一句带过。
8. 均未涉及，标注“双方均未涉及，无差距”。

## 全局规则

- 单证据单归属、单次使用；归属由位置和作用共同决定，不由证据类型决定。
- 证据三对齐：口播证据、画面截图、画面证据必须指向同一结论。
- 套装视频按整套产品判断，卖点维度看组合价值和划算感。
- 以目标市场文化视角评判，不用中文默认视角误伤本地有效手法。
- 若提供市场背景包，严格遵守其中红线。

## 输出侧要求

- 总结论按权重收敛，不要把低权重调性问题排到 Hook / 卖点 / CTA 之前。
- 关键结论不超过 5 点，按 GMV 影响排序。
- 亮点单列，用正向措辞，如“这点你做得比标杆好”。
- 优先级只用“优先改进”和“其次优化”，避免“必改”等过重措辞。
